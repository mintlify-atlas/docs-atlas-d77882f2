---
title: "Get Tikera running locally in minutes"
description: "Clone and run Tikera locally in a few steps — set up the Laravel backend and React frontend to start exploring the cinema ticketing app."
---

Tikera is a full-stack cinema ticketing application built with React on the frontend and Laravel on the backend. It lets users browse movies, view screenings, select seats, and book tickets — while administrators can manage movies and screenings through a dedicated admin panel. This guide walks you through cloning the repository and running both servers locally.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js** v16 or higher
- **PHP** 8.1 or higher
- **Composer**

## Setup

<Steps>
  <Step title="Clone the repository">
    Clone the Tikera repository and navigate into the project root:

    ```bash
    git clone https://github.com/raczkodavid/tikera-v2.git
    cd tikera-v2
    ```
  </Step>

  <Step title="Install frontend dependencies">
    Move into the `client` directory and install the Node.js dependencies:

    ```bash
    cd client
    npm install
    ```
  </Step>

  <Step title="Install backend dependencies">
    Move into the `server` directory and install the PHP dependencies via Composer:

    ```bash
    cd ../server
    composer install
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file to create your local `.env`:

    ```bash
    cp .env.example .env
    ```
  </Step>

  <Step title="Generate application key">
    Generate the Laravel application key:

    ```bash
    php artisan key:generate
    ```
  </Step>

  <Step title="Run migrations and seed the database">
    Run the database migrations and seed initial data (including the admin user):

    ```bash
    php artisan migrate --seed
    ```
  </Step>

  <Step title="Start the backend server">
    Start the Laravel development server from the `server` directory:

    ```bash
    php artisan serve
    ```

    The API will be available at `http://127.0.0.1:8000` by default.
  </Step>

  <Step title="Start the frontend dev server">
    In a separate terminal, start the Vite development server from the `client` directory:

    ```bash
    cd client
    npm run dev
    ```

    The frontend will be available at `http://localhost:5173` by default.
  </Step>
</Steps>

<Note>
  Want to explore Tikera without setting up locally? A live demo is available at [https://tikera-v2-ten.vercel.app/](https://tikera-v2-ten.vercel.app/). To access the admin panel, sign in with **admin@example.com** and the password **admin**.
</Note>
